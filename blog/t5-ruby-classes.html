Classes

March 29th 2015 / Technical Blog

Classes are like groups. Classes allow you to define methods and variables inside of them and reuse them across multiple objects.

Typically you would write a new class like so.


  class Person
    def name
      name = "Rob"
    end
  end

You are then able to spawn new instances of that class object. These new objects have access to the methods and variables defined in the Class.


  # Create new object which is an instance of the Person Class
  new_customer = Person.new
  p new_customer.name #=> "Rob"

So far, pretty easy right? You should be able to see the potenial of classes for everyday applications.

Let's add the ability to change the name variable and see what happens


class Person
  @first_name = ""
  def first_name
    @first_name
  end
  def change_first_name(name)
    @first_name = name
  end
end

new_customer = Person.new
p new_customer.first_name #=> nil
new_customer.change_name("Bob")
p new_customer.first_name #=> "Bob"

Now we've defined two methods in our class. One to display the first_name and the other to change it. We've also introduced a new type of variable @first_name which is called a class instance variable. The local variable we were using did not have enough scope to talk between the seperate methods #first_name and #change_first_name.

There is also a third variable called a class variable which looks like @@first_name. Class variables are inherited by new class instances and class instances of descendant classes. But if you change a class variable anywhere it will change everywhere, which may not be desirable.

Other important concepts that go with Classes are attributes, understanding class variables vs class instance variable scope and inheritence, understanding context and self and how that applies to class objects and the main class.
