Ruby Hashes VS JavaScript Objects

Apr 12th 2015 / Technical Blog

This week we learnt beginner JavaScript. Initially going to JavaScript from Ruby was a bit of a jolt. But I think that's because I'd just got used to the Ruby syntax.

In JavaScript the workhorses seem to be functions and objects. JavaScript has only 1 variable type, and it's global, so you rely more on scope.

Some of the obvious syntax differences in JavaScript are that most functions need to wrapped in parenthesis () and you need to terminate lines with semicolons ;

Ruby Hashes are similar to JavaScript Objects. JavaScript objects are declared as variables and contain key/value pairs.

In Ruby hashes were the used occasionally, and arrays were more common. Ruby hashes often seemed troublesome to access.

From my limited experience with JavaScript thus far it seems that JavaScript objects are commonly used. It may be that defining nested objects in JavaScript is clearer and adding, removing and accessing parameters just seems more natural. So we naturally use them more often.


ruby_hash = { 
  "a" => 1,
  "b" => 1, 
  "c" => 2 
}

var javascriptHash = { 
  a: 1,
  b: 2,
  c: 3
}

What I liked about the JavaScript object was the ease in which you can create and read a nested data structure.


var athlete = {
  robert = {
    name: "Robert",
    height: 190,
    sport: "Baseball",
    quote: "Imma get you boy",
    win: ""
  }
  chris = {
    name: "Chris",
    height: 170,
    sport: "Baseball",
    quote: "cant touch this",
    win: ""
  }
}

Adding, removing or updating parameters in a Javascript is a breeze and has a nice syntax


// Update the sport for the Athlete Robert
athlete.robert.sport = "Bob";
// Add a new paramter to the Athlete chris
athlete.chris.eyecolor = "brown";
